DESCRIPCIÓN DEL PROYECTO

El presente proyecto consiste en la implementación de una base de datos orientada a grafos utilizando Neo4j y el lenguaje Cypher. El sistema modela un entorno turístico inteligente mediante nodos y relaciones que representan usuarios, lugares, ciudades, eventos, alojamientos e intereses.

Para el desarrollo del modelo se construyó un conjunto de datos con múltiples registros por cada componente de la base de datos, permitiendo la realización de consultas complejas y sistemas de recomendación basados en afinidades, intereses, historial de visitas y conexiones entre usuarios.

El objetivo principal es comprender el funcionamiento de las bases de datos orientadas a grafos y aplicar técnicas de modelado y análisis utilizando consultas Cypher.

CONSULTA 1

Objetivo:
Identificar los intereses que más definen los lugares visitados por un usuario específico.

Código:

MATCH (u:Usuario {nombre:'Juan Pérez'})-[:VISITÓ]->(l:Lugar)
MATCH (l)-[:DEFINE_ESTILO]->(i:Interes)
RETURN i.nombre AS interes, COUNT(i) AS cantidad
ORDER BY cantidad DESC

------------------------------------------------------------

CONSULTA 2

Objetivo:
Consultar los lugares visitados y favoritos de un turista en una ciudad específica.

Código:

MATCH (u:Usuario {nombre:'Juan Pérez'})-[v:VISITÓ]->(l:Lugar)
MATCH (l)-[:SITUADO_EN]->(c:Ciudad {nombre:'Cali'})
RETURN l.nombre AS lugar,
v.es_favorito AS favorito,
c.nombre AS ciudad

------------------------------------------------------------

CONSULTA 3

Objetivo:
Identificar los lugares más populares de una ciudad.

Código:

MATCH (u:Usuario)-[:VISITÓ]->(l:Lugar)
MATCH (l)-[:SITUADO_EN]->(c:Ciudad {nombre:'Cali'})
RETURN l.nombre AS lugar,
COUNT(u) AS total_visitas
ORDER BY total_visitas DESC

------------------------------------------------------------

CONSULTA 4

Objetivo:
Consultar los alojamientos con presencia en más de tres ciudades.

Código:

MATCH (a:Alojamiento)-[:SITUADO_EN]->(c:Ciudad)

WITH a, COUNT(c) AS total_ciudades

WHERE total_ciudades > 3

RETURN a.nombre AS alojamiento,
total_ciudades

------------------------------------------------------------

CONSULTA 5

Objetivo:
Recomendar lugares según intereses de sitios visitados previamente.

Código:

MATCH (u:Usuario {nombre:'Juan Pérez'})-[:VISITÓ]->(l1:Lugar)
MATCH (l1)-[:DEFINE_ESTILO]->(i:Interes)
MATCH (l2:Lugar)-[:DEFINE_ESTILO]->(i)

WHERE NOT (u)-[:VISITÓ]->(l2)
AND l1 <> l2
AND i.nombre = '#Gourmet'

RETURN DISTINCT
l2.nombre AS lugar_recomendado,
i.nombre AS interes

------------------------------------------------------------

CONSULTA 6

Objetivo:
Recomendar eventos según afinidades del usuario.

Código:

MATCH (u:Usuario {nombre:'Juan Pérez'})-[a:AFÍN_A]->(i:Interes)
MATCH (e:Evento)-[:PROMUEVE]->(i)

WHERE e.sold_out = false
AND i.nombre = '#Gourmet'

RETURN
e.nombre AS evento,
i.nombre AS interes,
a.nivel_afinidad AS afinidad

ORDER BY afinidad DESC

------------------------------------------------------------

CONSULTA 7

Objetivo:
Recomendar alojamientos según experiencias de amigos.

Código:

MATCH (u:Usuario {nombre:'Juan Pérez'})-[:CONOCE_A]->(amigo:Usuario)
MATCH (amigo)-[:SE_HOSPEDÓ]->(a:Alojamiento)
MATCH (a)-[:SITUADO_EN]->(c:Ciudad {nombre:'Cali'})

WHERE NOT (u)-[:SE_HOSPEDÓ]->(a)

RETURN DISTINCT
a.nombre AS alojamiento,
c.nombre AS ciudad
